### 遗留问题：LLM 工具调用 JSON 参数包含 HTML 时解析失败
**现象**：LLM 生成 tool_calls 时，arguments 内嵌 HTML 内容，引号嵌套导致 `nlohmann::json::parse(arguments_json)` 失败。

**根因分析（第一性原理）**：
1. LLM 生成: `"arguments": "{\"content\": \"<div class=\\\"foo\\\">\"}"`
2. 外层 nlohmann 解析后 arguments 字符串: `{"content": "<div class=\"foo\">"}`
3. 内层 `nlohmann::json::parse(arguments_json)` 看到 `"foo"` 中引号未转义 → parse 失败

**数据流追踪**：
- `call_llm()` → `sanitize_json_recursive(reply)` → `tc["function"]["arguments"]` 提取为 string
- `execute_tool()` → `nlohmann::json::parse(arguments_json)` → **失败点**
- sanitize_json_string 只处理控制字符，对 `"` 和 `\` 透传，不改变转义

**修复方向**：
- 在 `execute_tool` 中 try-catch JSON parse 失败
- 失败时应用 JSON string repair：状态机追踪字符串边界，对字符串内非结构化的 `"` 进行转义
- 不在 cs_output 侧改，不引入 base64，保持最小改动
